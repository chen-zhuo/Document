# Basis

### 简介

> 好记性不如烂笔头。——《俗语》

> 工欲善其事，必先利其器。——《论语·卫灵公》

​		一个人在学习的过程中，总少不了记笔记，而且我一直认为自己记笔记才是最高效的学习方式，看别人的笔记远不如自己写笔记印象来的深刻，所以自己动手来笔记吧！

​		作为一个程序员，用笔的时候肯定没有用电脑的时候多，因此我也更愿意把 “写笔记” 称之为 “写文档”。因此本章节将会介绍：快捷键、文档写作工具 *Typora*、项目托管平台 *GitHub*、静态展示网站 *GitHubPage*、版本控制 *Git*。

### *Typora*

[*Typora*](https://www.typora.io/) 是一款轻便简洁的 *Markdown* 编辑器。

**优点**：支持即时渲染技术、支持 *Markdown​* 语法、支持 *LaTeX* 语法。

**缺点**：文件内容过多会无法打开，不支持图片编辑。

### *GitHub*

[*GitHub*](https://github.com/) 是一个托管项目代码的平台。

**优点**：开发者人数众多，免费的开源仓库，支持 *Markdown* 语法。

**缺点**：不支持 *LaTeX* 语法。

### *GitHubPage*

[*GitHubPage*](https://pages.github.com/) 是为 *GitHub* 账户开设的网站。

**优点**：免费的静态网站，搭建步骤简单，多种主题可自由选择。

**缺点**： 不能自动读写站内的文件，不能运行 *PHP*。

### *Git*

[*Git*](https://git-scm.com/) 是一个开源的分布式版本控制系统。

**优点**：可以有效、高速地处理项目版本管理，可以很方便管理 *GitHub* 上的仓库。

**缺点**：命令行操作略显枯燥，不够直观。
